Goal:

Requirements:

 - session_id
 - user query
 - timestamp
 - num_documents_in_session
 - retrieved_chunks_count (after threshold)
 - retrieval_time
 - generation_time
 - total_time

Log per retrieved chunk:
 - query_id
 - rank (1 = most similar)
 - chunk_id
 - document_id
 - doc_title
 - doc_source
 - page_number
 - similarity_score
 - chunk_length (tokens/characters)

✅ Stage 2 — Gold Evaluation Dataset

Goal:
 - Create a ground truth dataset to measure correctness.

Requirements:
 - For each book/session, create 50–100 representative questions.
 - Each entry should include:
 - question_text
 - ground_truth_answer (manual extraction from book)
 - source_chunk_ids (optional, but ideal)
 - source_page_numbers (optional)
 - Store in CSV: evaluation_dataset.csv

Tips:
 - Mix easy factual questions + conceptual / narrative questions.
 - Include multi-sentence answers, since books have context.
 - Keep question diversity across your uploaded books.

Your Task (Stage 2)

✅ Stage 3 — Retrieval Evaluation (Expanded)
Core (what you already planned)

✔ Hit@k
✔ Recall@k
✔ MRR
✔ Similarity distribution
✔ Retrieved count

Optional Richer Metrics (if you want deeper insight)

✔ Precision@k — Among top-k retrieved chunks, how many are relevant?
✔ F1@k — Harmonic mean of precision and recall
✔ nDCG@k — Rewards correct retrieval higher in ranking
✔ MAP — Averages precision across ranking positions

👉 You can compute these manually if you annotate whether each chunk is relevant or not.
